# ThreePipe 珠宝展示项目问题排查与解决总结

# 项目概览

基于 React \+ ThreePipe 构建的3D珠宝展示应用，开发中遇到环境配置、依赖安装、模型加载、渲染优化等问题，本文档总结核心排查路径与解决方案，供后续开发维护参考。

# 一、环境与依赖问题

## 1\.1 依赖安装与路径解析

- **问题现象**：运行报错，Vite无法解析@threepipe相关插件及Three\.js旧版材质扩展模块路径

- **问题根源**：缺失@threepipe相关依赖；Three\.js r150\+ 移除旧版材质扩展原生支持

- **解决方案**：安装缺失依赖（npm install @threepipe/webgi\-plugins @threepipe/plugin\-tweakpane）；删除旧扩展导入语句，通过模型转换解决兼容性

# 二、模型与渲染问题

2.1 模型加载失败（黑屏）与兼容性
问题现象：GLTF/GLB 模型渲染全黑，控制台报未知材质扩展错误
问题根源：模型使用旧版 Specular-Glossiness PBR 材质工作流；模型路径引用错误
解决方案：用 gltf-transform 转换材质工作流；修正模型加载路径，统一放入 public/models/ 目录用绝对路径加载

2.2 模型渲染为黑色（无光照）
问题现象：模型加载成功但无材质细节，呈黑色
问题根源：场景未添加光源，PBR 材质依赖光照计算外观

解决方案：viewer.load () 成功后，添加环境光和方向光
2.3 Canvas 渲染性能与显示异常
问题现象：页面有水平滚动条，控制台报 WebGL 尺寸相关错误
问题根源：Tweakpane UI 面板无宽度限制；Canvas 尺寸因设备像素比放大，超出 GPU 最大纹理尺寸
解决方案：为 UI 面板添加 CSS 宽度限制；显式设置 Canvas 的 width/height 属性，通过样式控制显示尺寸

2.4 自定义后处理着色器模块路径找不到
问题现象：运行报错，无法找到 / 解析 EffectComposer、ShaderPass、RenderPass 等后处理模块,
问题根源：Three.js 后处理相关模块导入路径错误，未匹配当前项目 Three.js 源码目录结构
解决方案：
场景渲染 → 离屏帧缓冲区(FrameBuffer)存屏幕纹理 → 高亮提取 → 多级高斯模糊生成光晕 → 叠加回原图输出没有向外扩散的朦胧光晕，所以肉眼几乎看不出生效。
对照 Three.js 源码目录结构修正导入路径，统一使用 three/examples/jsm/postprocessing/ 标准路径导入后处理模块

# 三、核心总结

1. 模型兼容性：需检查下载模型的材质工作流，用gltf\-transform预处理，确保与Three\.js版本兼容

2. 光照配置：PBR模型必须正确添加光源和环境贴图，否则无法正常显示

3. 性能管控：合理限制Canvas像素尺寸与显示尺寸，约束第三方UI插件布局，避免超出GPU限制

4. 插件使用：优先采用ThreePipe官方插件实现后期特效，集成度和性能更优

通过上述排查与解决，应用实现稳定运行，达成预期视觉与交互效果。
